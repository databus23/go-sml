# go-sml Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers-extended-cc:subagent-driven-development (if subagents available) or superpowers-extended-cc:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build an idiomatic Go library for parsing SML (Smart Message Language) data from German smart meters.

**Architecture:** Root `sml` package with types, TLV decoder, message parsers, convenience methods, and streaming API. Separate `transport` sub-package for frame detection from io.Reader. Parse-only, no serialization.

**Tech Stack:** Go 1.22+, stdlib only, zero external dependencies.

**Spec:** `docs/superpowers/specs/2026-04-08-go-sml-design.md`

**Reference C implementation:** https://github.com/volkszaehler/libsml

### Task 1: TLV Decoder — Core Byte Reading

**Files:** Create `decode.go`, `decode_test.go`

The TLV decoder is the foundation. SML encodes everything as type-length-value fields. Reference: libsml `sml_shared.c`.

**SML TLV format:**
- Each element starts with 1+ type-length (TL) bytes
- First TL byte: bit 7 = "more" flag, bits 6-4 = type (0x0=octet string, 0x4=boolean, 0x5=signed int, 0x6=unsigned int, 0x7=list), bits 3-0 = length nibble
- If "more" bit set, next byte provides additional length bits (shift left 4, OR in)
- For non-list types: length includes the TL byte(s) themselves, so data length = length - number_of_TL_bytes
- For list types: length IS the element count directly
- Special bytes: 0x00 = end of SML message, 0x01 = optional field absent
- Numbers are big-endian, variable width (1/2/4/8 bytes), sign-extended for signed types

**decoder struct:**
- `buf []byte`, `pos int`, `err error` (sticky error pattern)
- All methods are no-ops once `err` is set

- [ ] Write failing tests for `decoder` in `decode_test.go`:
  - `TestDecoderReadByte`: read single byte, advance pos, error on EOF
  - `TestDecoderReadTypeLength`: single-byte TL (e.g., `0x65` = unsigned, length 5 → type=0x60, data_len=4), multi-byte TL with "more" bit
  - `TestDecoderIsEndOfMessage`: byte 0x00 returns true without consuming
  - `TestDecoderIsOptionalSkipped`: byte 0x01 returns true and consumes it
  - `TestDecoderStickyError`: after error, subsequent reads are no-ops
- [ ] Run tests, verify they fail
- [ ] Implement `decoder` struct in `decode.go` with: `newDecoder([]byte)`, `readByte()`, `readTypeLength() (typ byte, length int)`, `isEndOfMessage() bool`, `isOptionalSkipped() bool`, sticky error handling

### Task 7: Top-Level Decode and DecodeWithOptions

**Files:** Create `sml.go`, modify `sml_test.go`

The entry point that ties everything together. An SML file is a sequence of messages. Each message is a 6-element list: transaction_id, group_id, abort_on_error, message_body (2-element list: tag + data), crc, end_of_msg (0x00).

**Message body dispatch:** Read the u32 tag, switch on it:
- 0x00000101 → readOpenResponse
- 0x00000201 → readCloseResponse
- 0x00000701 → readGetListResponse
- 0x00000501 → readGetProcParameterResponse
- 0x00000401 → readGetProfileListResponse
- 0x00000301 → readGetProfilePackResponse
- 0x0000FF01 → readAttentionResponse
- Request tags (even numbers) → skip the message body

Between messages, skip trailing 0x00 bytes (end-of-message markers).

**DecodeWithOptions:** When Strict=false, catch errors per-message, continue parsing, return partial File + errors.Join.

### Task 8: CRC-16 Implementations

**Files:** Create `transport/crc.go`, `transport/crc_test.go`

Two CRC-16 variants needed. Both use the same reflected lookup table (polynomial 0x8408, CRC-CCITT reflected).

**CRC-16 X.25 (standard, DIN 62056-46):**
- Init: 0xFFFF
- Iteration: `crc = (crc >> 8) ^ table[(crc ^ byte) & 0xFF]`
- Final: XOR with 0xFFFF, then byte-swap (little-endian to big-endian)

**CRC-16/Kermit (Holley workaround):**
- Init: 0x0000
- Same iteration
- Final: no XOR, no byte-swap

First 16 table entries for verification: 0x0000, 0x1189, 0x2312, 0x329b, 0x4624, 0x57ad, 0x6536, 0x74bf, 0x8c48, 0x9dc1, 0xaf5a, 0xbed3, 0xca6c, 0xdbe5, 0xe97e, 0xf8f7

- [ ] Write failing tests:
  - `TestCRC16Table`: verify first 16 entries match expected values
  - `TestCRC16`: known input → known CRC (use the SML start sequence `1b1b1b1b 01010101` as test input, compute expected CRC by hand or from C library)
  - `TestCRC16Kermit`: same input, different expected CRC
  - `TestCRC16EmptyInput`: edge case
- [ ] Run tests, verify they fail
- [ ] Implement `CRC16(data []byte) uint16` and `CRC16Kermit(data []byte) uint16` in `transport/crc.go`. Generate the full 256-entry lookup table.

### Task 9: Transport Reader

**Files:** Create `transport/transport.go`, `transport/transport_test.go`

State machine that reads SML frames from an io.Reader.

**States:** scanningForStart → readingPayload → validating

**Start detection:** Scan for 8 consecutive bytes: `1b 1b 1b 1b 01 01 01 01`

**Payload reading:** Read 4 bytes at a time. If 4 bytes are `1b 1b 1b 1b`, peek at next 4 bytes:
- If next byte is `1a`: end of frame. Remaining bytes are: pad_count (1), CRC (2).
- If next 4 bytes are `1b 1b 1b 1b`: escape sequence. Emit literal `1b 1b 1b 1b` to payload, continue.

**CRC validation:** Compute CRC over: start_sequence + payload + padding + end_sequence + pad_count_byte. Try standard CRC16 first, then Kermit. Both must match the 2 CRC bytes (big-endian).

**Strip padding:** Remove `pad_count` bytes from end of payload.

**Max frame size:** 64KB default. Error if exceeded.

- [ ] Write failing tests:
  - `TestReaderSimpleFrame`: construct a valid SML frame (start + some bytes + end + CRC), verify Next() returns the payload
  - `TestReaderEscapeSequence`: frame with `1b1b1b1b` in payload (doubled on wire), verify it's unescaped
  - `TestReaderPadding`: frame with 1-3 padding bytes, verify they're stripped
  - `TestReaderMultipleFrames`: two frames in sequence, verify Next() returns each
  - `TestReaderGarbageBeforeStart`: random bytes before start sequence, verify they're skipped
  - `TestReaderCRCFailure`: frame with wrong CRC → error
  - `TestReaderKermitCRC`: frame with Kermit CRC (not standard) → still accepted
  - `TestReaderMaxFrameSize`: oversized frame → error
  - `TestReaderEOF`: reader returns io.EOF → Next() returns io.EOF
- [ ] Run tests, verify they fail
- [ ] Implement Reader in `transport/transport.go`
